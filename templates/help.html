Help

Creating a Session

Anybody can create a new session from the homepage.

The user is presented with a link to the session and a link to the admin page for the session. The admin page is only secured with a random string. Anyone with the link to the admin page will be able to moderate a session.

It's important for the user who creates the session to note the URL for the admin page. If this is lost then an overall server admin will need to retrieve it from the Server Admin page.

Asking a Question

Anyone with a link to the session can ask a question. Questions are anonymous. Users can also add upvotes to a question. Users can only upvote a question once, but this is only enforced through a session cookie. There's nothing stopping a user from using multiple browsers or incognito tabs to vote multiple times. Hopefully people can be trusted.

Moderating a Session

A user can moderate a session in the following ways:

  • Hide a question once it's been answered
  • Unhide a question if it needs to come back
  • Close a session so no more questions can be asked
  • Export the list of questions
  • Delete a session when it has concluded

Anyone who has the session admin URL can moderate a session.

Closing a Session

Users will see a message if the session is closed and will not be able to add any more questions. They will still be able to add upvotes.

Deleting a Session

Once a session is deleted no one will have access to it or its content. You need to export the questions before deleting the session if you want to keep the questions.

Server Admin

Server admins can see all current sessions, including those that have been closed. Server admins can provide links to any session, including admin links.

Anyone who has the session admin URL can moderate a session.
This means server admins can administer any current session.
